IT MUST ADHERE TO ALL HUMAN COMPUTER INTERACTION STANDARDS, INCLUDING Jakob Nielsen’s 10 usability HEURISTICS(1. Visibility of system status
The design should always keep users informed about what is going on, through appropriate feedback within a reasonable amount of time.
Example: A progress bar showing how much of a file has been uploaded.
2. Match between system and the real world
The design should speak the users' language. Use words, phrases, and concepts familiar to the user, rather than internal jargon.
Example: Using a "trash can" icon for deleting files.
3. User control and freedom
Users often perform actions by mistake. They need a clearly marked "emergency exit" to leave the unwanted action without having to go through an extended process.
Example: An "Undo" button after accidentally deleting an email.
4. Consistency and standards
Users should not have to wonder whether different words, situations, or actions mean the same thing. Follow platform and industry conventions.
Example: Placing the "Settings" icon as a gear shape, as users expect this.
5. Error prevention
Good error messages are important, but the best designs carefully prevent problems from occurring in the first place.
Example: Graying out a "Submit" button until all required fields are filled correctly.
6. Recognition rather than recall
Minimize the user's memory load by making elements, actions, and options visible. The user should not have to remember information from one part of the interface to another.
Example: A "Recently Viewed" section in an e-commerce store.
7. Flexibility and efficiency of use
Shortcuts — hidden from novice users — may speed up the interaction for the expert user such that the design can cater to both inexperienced and experienced users.
Example: Keyboard shortcuts (Ctrl+C, Ctrl+V) for power users.
8. Aesthetic and minimalist design
Interfaces should not contain information which is irrelevant or rarely needed. Every extra unit of information competes with the relevant units of information.
Example: A clean Google search homepage vs. a cluttered news portal.
9. Help users recognize, diagnose, and recover from errors
Error messages should be expressed in plain language (no error codes), precisely indicate the problem, and constructively suggest a solution.
Example: Instead of "Error 404", saying "The page you are looking for does not exist. Try going back to the Home page."
10. Help and documentation
It’s best if the system doesn’t need any additional explanation. However, it may be necessary to provide documentation to help users understand how to complete their tasks.
Example: A searchable "Help Center" or tooltips explaining complex features.
